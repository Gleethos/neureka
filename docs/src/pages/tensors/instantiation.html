Tensor instantiation:

At the core of Neureka lies the `Tsr` class,
which is a three-letter abbreviation of the word tensor!

The most readable and type safe way of instantiating a `Tsr` is by using the internal tensor builder which is exposed by the static `Tsr::of(T type)` factory method. 


var t = Tsr.of(Double)
                .withShape(2, 4)
                .andFill(
                    1, 2, 3, 4,
                    9, 8, 6, 5
                )
                

Tsr<Double> t = Tsr.of(Double.class)
                    .withShape(2, 4)
                    .andFill(
                        1, 2, 3, 4,
                        9, 8, 6, 5
                    );
                

val t = Tsr.of(Double::class.java)
                .withShape(2, 4)
                .andFill(
                    1.0, 2.0, 3.0, 4.0,
                    9.0, 8.0, 6.0, 5.0
                );
                

var t = Tsr.of([2, 4],[
            1, 2, 3, 4,
            9, 8, 6, 5
        ])
                

Tsr<Double> t = Tsr.of(
             new int[]{2, 4},
             new double[]{
                    1, 2, 3, 4,
                    9, 8, 6, 5,
            });
                

val t = Tsr.of(
     intArrayOf(2, 4),
     doubleArrayOf(
            1, 2, 3, 4,
            9, 8, 6, 5,
    ))

In this example the first argument represents the shape of the data and the second argument the data itself. Both arguments can be either a List or an Array. 


var t = Tsr.of([2, 4], -5.0)
                

Tsr<Double> t = Tsr.of(new int[]{2, 4}, -5d);
                

val t = Tsr.of(intArrayOf(2, 4), -5.0)

If you want to instantiate a tensor filled homogeneously with a given value, simply replace the latter argument with a double, float, int or any other numeric type. 


var t = Tsr.of(-5)
                

Tsr<Integer> t = Tsr.of(-5);
                

val t = Tsr.of(-5)

Not passing any shape information to the constructor will simply lead to the creation of a scalar tensor.
