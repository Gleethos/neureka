Device instantiation:

Neureka has one common interface for many device implementations. Therefore the instantiation of a device is also dependent on its implementation. The most commonly used Device type is the OpenCLDevice! However when one is not willing to read about the different types Neureka provides one can simply query devices via the following method : 



    def device = Device.find('first gpu')

                


    Device device = Device.find("first gpu");

There are multiple things that can put into this query string. It is for example also possible to simply pass the name of the device itself. Even if the name of the device contains spelling errors Neureka will try to interpret what you mean. If however it cannot find what you are looking for it will simply return null.

If this level of control for device selection for your tensors is too shallow then you can look at concrete implementations : 



    def platforms = OpenCLPlatform.PLATFORMS()
    def devices = platforms.get(0).devices.get(0)

                


    List platforms = OpenCLPlatform.PLATFORMS();
    List devices = platforms.get(0).getDevices().get(0);

OpenCLDevice instances are instantiated during class loading. This is because a single instance of this class should only represent a single real device.

Devices are components of tensors and tensors are elements of devices. Therefore the following two options migrate tensors to a given device: 


    myTensor.set( device )
                

    myTensor.set( device );
                

    device.store( myTensor )
                

    device.store( myTensor );
