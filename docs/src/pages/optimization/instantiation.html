Optimization

The optimization package provides the `Optimization` type, a functional interface for implementing optimization algorithms. Neureka offers a number of implementations, based on the `Optimizer` type, which is a subtype of `Optimization`. This `Optimizer` interface also exposes a simple API for configuring and creating the most common optimization algorithms. It is important to note that optimizations are `Tensor` components. So a tensor will use an optimizer when its gradients should be applied (see `Tensor::applyGradients()`). Here an example of how to use one of the most basic optimization algorithms, *stochastic gradient descent*:


var w = Tensor.of(0f)
w.set(Optimizer.SGD.withLearningRate(0.01)) // 0.01 learning rate
                

Tensor<Float> w = Tensor.of(0f);
w.set(Optimizer.SGD.withLearningRate(0.01)); // 0.01 learning rate
                

val w = Tensor.of(0f)
w.set(Optimizer.SGD.withLearningRate(0.01)) // 0.01 learning rate

An optimizer can easily be accessed by calling 'find(Optimizer.class)' on the tensor of concern.

It is important to note: Every tensor has their own `Optimizer` instance. It is therefore not allowed to be shared due to the fact that the `Optimizer` stores a context specialized to the one given tensor. 


var w = Tensor.of(0)
w.set(Optimizer.ADAM)
println w.find(ADAM).isPresent() // true
                

var w = Tensor.of(0);
w.set(Optimizer.ADAM);
System.out.println(w.find(ADAM.class).isPresent()); // true
                

val w = Tensor.of(0)
w.set(Optimizer.ADAM)
println(w.find(ADAM::class.java).isPresent()) // true
