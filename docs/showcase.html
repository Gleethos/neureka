Features

Slicing:

Just as in libraries like NumPy subsets of tensors or 'nd-arrays' can by extracted while retaining the same underlying data. This powerful feature of Neureka is showcased via some groovy code below! 



        Tsr a = new Tsr([4, 6], [
                1, 2, 3, 4,
                5, 6, 7, 8,
                9, 1, 2, 3,
                4, 5, 6, 7,
                8, 9, 1, 2,
                3, 4, 5, 6
        ])

        // Slice with offsets:
        b = a[1, -2] // [2.0, 3.0, 4.0, 6.0, 7.0, 8.0, 1.0, 2.0, 3.0, 5.0, 6.0, 7.0]
        //
        //    1, 2, 3, 4, => 2, 3, 4,
        //    5, 6, 7, 8, => 6, 7, 8,
        //    9, 1, 2, 3, => 1, 2, 3,
        //    4, 5, 6, 7, => 5, 6, 7,
        //    8, 9, 1, 2,
        //    3, 4, 5, 6
        //

        // Note: The underlying data of a will be inherited by b! :D
        a.value64()[1] = a.value64()[1] * 6
        a.value64()[7] = a.value64()[7] * 2
        //>> b's values would then be: [12.0, 3.0, 4.0, 6.0, 7.0, 16.0, 1.0, 2.0, 3.0, 5.0, 6.0, 7.0]

        //Use b for calculations but keep in mind that it affects a! :
        Tsr c = new Tsr([3, 4], [
                -3, 2, 3,
                 5, 6, 2,
                -1, 1, 2,
                 3, 4, 2,
        ])
        Tsr d = b + c // b can be used for any kind of calculation
        //
        //    9,  5,  7,
        //    11, 13, 18,
        //    0,  3,  5,
        //    8,  10, 9
        //

Using Groovy's range operator:

Groovys '..' operator is equivalent to Pythons ':'. In NumPy this operator is used to extract subsets of nd arrays. Here is the equivalent in Neureka: 



        // Use ranges (The '..' operator is similar to pythons ':' operator)
        b = a[1..3, 2..4] // [3x3][1.0, 2.0, 3.0, 5.0, 6.0, 7.0, 9.0, 1.0, 2.0]
        //
        //    1, 12, 3, 4,
        //    5, 6, 7, 16,
        //    9, 1, 2, 3,  =>  1, 2, 3,
        //    4, 5, 6, 7,  =>  5, 6, 7,
        //    8, 9, 1, 2,  =>  9, 1, 2,
        //    3, 4, 5, 6
        //

Subsets / Slices with steps:

Within the given range (which defines the subset location) a 'scale'- or 'step'-size can be chosen for every dimension. The dimension of the output tensor will then have a size which is calculated according to the number of steps which fit into the subset space: 



        //  '[i..j]:k' is equivalent to pythons 'i:j:k' syntax for indexing! (numpy)
        //  i.. start index.
        //  j.. end index. (inclusive!)
        //  k.. step size.
        b = a[[[0..3]:2, [1..4]:2]] // [2x2][5.0, 7.0, 4.0, 6.0]
        //
        //   1, 12, 3, 4,
        //   5, 6, 7, 16,  =>  5,  7,
        //   9, 1, 2, 3,
        //   4, 5, 6, 7,  =>  4,  6,
        //   8, 9, 1, 2,
        //   3, 4, 5, 6
        //

Introduction

Neureka implements so called Auto-Differentiation. AD-algorithms are the backbone of today's most advanced artificial neural networks. It allows for calculating derivatives with respect to weight variables used to correct errors incrementally throughout many training iterations whereby neural networks then gain the ability to approximate, generalize and thereby learning. Differentiation goes both ways: Forwards and Backwards. Namely:
Forward-mode-AD and backward-mode-AD.
Neureka utilizes both techniques to get the best performance. Especially the use of backward-mode-AD revolutionized machine learning in recent decades. Gradient decent using backward-mode-ad is termed back-propagation and for most cases this approach accelerates gradient descent by orders of magnitudes compared to other techniques, making the difference between a model taking weeks instead of years to train. Beyond its use in machine learning, Auto-differentiation is a powerful computational tool in a multitude of other areas, ranging from weather forecasting to analyzing numerical stability. Besides it's use in machine learning, AD is a common tool for many other fields dependent on algebraic numerical computation. 


    Tsr x = new Tsr(3).setRqsGradient(true);
    Tsr b = new Tsr(-4);
    Tsr w = new Tsr(2);
    Tsr y = new Tsr(new Tsr[]{x, b, w}, "((i0+i1)*i2)^2");
    /* *
     *   f(x) = ((x-4)*2)^2; :=>  f(3) = 4
     *   f(x)' = 8*x - 32 ;  :=>  f(3)' = -8
     *
     *   y.toString(): "[1]:(4.0); ->d[1]:(-8.0), "
     * */

In this example Neureka calculated the derivative of y w.r.t x. Forward-AD is used due to the fact that no other dependencies need to be considered. By utilizying forward-AD Neureka greatly decreases Memory consumption as well as increase performance. 


     Tsr a = new Tsr(-3).setRqsGradient(true);
     Tsr b = new Tsr(4).setRqsGradient(true);
     Tsr c = new Tsr(2);
     Tsr d = new Tsr(Tsr[]{a, b, c},"(relu(i0*i1)+i1)/i2");
