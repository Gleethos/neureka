I have found it!

A lightweight open source platform independent machine learning library written in Java.

Written in Java and compatible with any JVM language, such as Groovy, Scala, Clojure or Kotlin. 100% Platform independent!

The library is completely open-source and free to use,
MIT-Licenced.

Features GPU acceleration via OpenCl. Therefore not tied to any hardware vendor!

But why? Isn't Java verbose?

Popular deep learning libraries use python due to it's elegant Syntax. Java on the other hand is considered verbose and bulky, so why would anybody ever build a deep learning library for Java? The answer is simple:
Nobody did!
This library is written for every JVM-language, namely: Groovy, Kotlin, Scala, and Jython just to name a few. Neureka can be integrated by any language which compiles to JVM bytecode! Take a look at some Groovy and Java side by side:

Both code snippets below express the following equations:
f(x) = ((x-4)*2)^2; :=> f(3) = 4
f(x)' = 8*x - 32 ; :=> f(3)' = -8


//Groovy:
Tsr x = new Tsr(3).setRqsGradient(true)
Tsr b = new Tsr(-4)
Tsr w = new Tsr(2)

Tsr y = ((x+b)*w)**2
// y: "[1]:(4.0); ->d[1]:(-8.0), "

Groovy supports operator overloading. Neureka implements certain methods which will be called when corresponding operators are used. Such language features allow for elegant and short implementations. For rapid prototyping and small scale projects groovy, scala or jython might be the perfect choice. 


//Java:
Tsr x = new Tsr(3).setRqsGradient(true);
Tsr b = new Tsr(-4);
Tsr w = new Tsr(2);
Tsr y = new Tsr(new Tsr[]{x, b, w}, "((i0+i1)*i2)^2");

// y: "[1]:(4.0); ->d[1]:(-8.0), "

...if however performance is of concern for you and your team then Neureka can easily be integrated with Java or Kotlin to allow for more efficient runtime.

If you prefer fast prototyping with Jupyter then Neureka can be used there too. BeakerX is a jupyter Kernel that supports many JVM languages like Groovy, Scala, Clojure, Kotlin and Java.

But isn't Java slow?

Python currently provides the most advanced environments and ecosystems for machine learning and data science. This is despite the fact that it is an interpreted language and thereby substantially slower than statically compiled languages and also allot slower than dynamically compiled and optimized bytecode executed during JVM runtime (JIT - compilation). Popular frameworks like PyTorch and Tensorflow compensate this deficit by utilizing C++ and Cuda internally. Neureka tries to achieve similar performance with a more balanced approach, where number crunching is offloaded to OpenCl whenever possible or otherwise JVM's advanced concurrency and JIT runtime optimizations come into play. (Which should not be underestimated)
