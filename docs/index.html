A lightweight open source platform independent tensor library written in Java.

Written in Java and compatible with any JVM language, such as Groovy, Scala, Clojure or Kotlin. 100% Platform independent!

The library is completely open-source
and free to use,

Features GPU acceleration via OpenCl. Therefore not tied to any hardware vendor!

Why Neureka?

Neureka trains your neural network using a computation graph recorder. This is contrary to the approaches found in other frameworks such as TensorFlow, Theano, Caffe, and CNTK which require the definition of a computation graph ahead of time. This means a developer has to build a neural network structure which cannot change during runtime. Neureka, uses the recorded computation graph in order to apply a technique called reverse-mode auto-differentiation, which allows your network structure to change during runtime arbitrarily with zero lag or overhead.
This powerful feature has been inspired by PyTorch. - [Motivation](docs/markdown/motivation.md) :fire:

Why Java?

Although Java is a robust and safe language, it is often times considered to be too verbose and explicit for simple prototyping or more explorative workloads... Therefore popular machine learning and tensor / deep learning libraries rely on python, which offers a very concise and simplistic syntax.
So one might come to wonder, why would anybody ever build a deep learning library for Java?
The answer is simple!

Nobody did!

This library was written for all JVM-languages, namely: Groovy, Kotlin, Scala, and Jython just to name a few.
Take a look at the following examples side by side!

Neureka can be integrated by any language which compiles to, or understands JVM bytecode! 



    Tsr x = Tsr.of(3).setRqsGradient(true)
    Tsr b = Tsr.of(-4)
    Tsr w = Tsr.of(2)

    Tsr y = ((x+b)*w)**2
    // y: "[1]:(4.0); ->d[1]:(-8.0), "

                


    Tsr x = Tsr.of(3).setRqsGradient(true);
    Tsr b = Tsr.of(-4);
    Tsr w = Tsr.of(2);
    Tsr y = Tsr.of(new Tsr[]{x, b, w}, "((i0+i1)*i2)^2");

    // y: "(1):[4.0]; ->d(1):[-8.0], "
Both code snippets express the following equations - `f(x) = ((x-4)*2)^2 | f(3) = 4` - `f(x)' = 8 * x - 32 | f(3)' = -8`

Some languages like Groovy for example supports operator overloading. Certain methods will be called when corresponding operators are used ( + : plus(...), - : minus(...), ... ). Such language features allow for elegant and short implementations. For rapid prototyping and small scale projects groovy, scala or jython might be the perfect choice.

...if however performance is of concern for you and your team, then Neureka can easily be used with Java or Kotlin to allow for a more efficient runtime.

If you prefer fast prototyping with Jupyter, then Neureka can be used there too.
BeakerX is a jupyter extension that supports many JVM languages like Groovy, Scala, Clojure, Kotlin and Java.

Performance

Neureka is a lightweight and highly modular library designed to allow for the support of any backend, if required. The operations within the default backend are implemented as generalized, modular and concise as possible, meaning that performance wise Neureka still needs to mature and would greatly appreciate improvement proposals. Its backend has a consistent API which can be extended easily by anyone willing to go the extra mile for more specialized operation kernels...

Besides that Neureka is also greatly held back by JVM's lack of allowing for more memory localized types and also a lack of an API for consistent SIMD vectorization.
(...take a look at the upcoming vector API...)

This vector API alongside the introduction of inline/value types from Project-Valhalla will greatly boost the development availability for ML on the JVM.
