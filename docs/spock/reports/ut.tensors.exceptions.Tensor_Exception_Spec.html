Tensors Exception Behavior
This specification covers the behavior of the $Tsr class in
    exceptional scenarios which are contrary to its intended use.
    The purpose of this is to assert that the $Tsr class will provide
    useful feedback to a user to explain that a misuse of its API
    occurred so that the user can correct this misuse.

Tensors Exception Behavior


This specification covers the behavior of tensors in exceptional situations.

Features:

  • Trying to inject an empty tensor into another causes fitting exception.
  • Passing null to various methods of the tensor API will throw exceptions.
  • Passing an invalid object into Tsr constructor causes descriptive exception.
  • Passing an invalid key object into the "getAt" method causes a descriptive exception.
  • Out of dimension bound causes descriptive exception!
  • Building a tensor with 0 shape arguments throws an exception.
  • Casting a tensor as something unusual will cuas an exception to be thrown.
  • Building a tensor with "null" as shape argument throws an exception.
